If this takes a long time, then consider trying some of the following to improve the performance:

  • In the Parser menu item / Edit Parameters:
    • Set the MaxPrefixes to the maximum number of prefixes and proclitics that a well-formed word could have.
    • Set the MaxInfixes to the maximum number of infixes that a well-formed word could have.
    • Set the MaxSuffixes to the maximum number of suffixes and enclitics that a well-formed word could have.
    • Set the MaxRoots to the maximum number of roots that a well-formed word could have.
    • Set the MaxNulls to the maximum number of null allomorphs that a well-formed word could have.
    • Set the MaxInterfixes to the maximum number of “interfixes” (affixes which can also appear between roots) that a well-formed word could have.
    • Set the MaxAnalyses to a small number. This limits the number of analyses that the parser will return (i.e. the parser will stop looking for more possibilities after it reaches this number). If this value is set to 0 or -1, then it will return all the analyses it finds.
  • Go to the Grammar area / Grammar Sketch and generate the sketch (click on the “Generate” button).
    • Examine the Residue section (assuming there is one). Try to flesh out any missing information indicated in the following sub-sections:
      • If there are any “Underspecified Derivational Affixes”, try and make them fully specified.
      • If there are any “Underspecified Inflectional Affixes”, try and make them fully specified.
      • If there are any “Unclassified Affixes”, try and figure out how to fully specify them (i.e. make them be derivational or inflectional with all the needed information specified).
      • If there are any “Unmarked Stems”, try and mark them with an appropriate category.
    • Examine the Compounding section. If it says that there is no compounding in your language and your language does have word-internal stem compounding, try adding compound rules.
