The current test bench app is called ImportExport and is located in the Samples folder in your branch.

The is no documentation for that app, of course, other than what I added to the app. Look at the [design view of ImportExport.cs], while I cover how it works.

The basics to get it working for a new BEP are:

Add the new BEP to the collection of BEPs that are pre-processed by creating and populating each from an xml source BEP data file.
The “Convert to all (Basic)” btn needs this, and that btn must be clicked before any other tests can be performed.

You will need to add two new radio btns. One goes in the “Source System” group and the other in the “Target System” group. The checked values of one or the other radio btn is used, when the “Load Source” or “Import” btns are clicked.

Having done the basic conversion in step 1, you can then select the number of repetitions the other three tests will do. There will always be one more than what you select, as that is done ‘for free’, and the times are not included in the output report. I tend to just go with the default of 10 reps for the full test loop, but I also start with more like 1, just to make sure it works.

Results come up as the average (mean) of the number of reps.

That is the overview. You’ll need to look at the current code to see how it works and how to add MySql to the test bench.
